# Фото

Фото — критичная часть замера. В приложении есть два сценария: online-фото сразу в Supabase и offline-фото в TEMP-черновике.

## Online-фото

Когда замер уже сохранён в Supabase и интернет доступен:

1. Пользователь выбирает тип фото.
2. Делает фото камерой или выбирает из галереи.
3. Приложение загружает файл в Supabase Storage bucket `measurement-photos`.
4. После загрузки создаётся строка в таблице `measurement_photos`.
5. Приложение перечитывает фото текущего замера и показывает превью.

Если замер ещё не сохранён, приложение сначала пытается сохранить черновик замера, а потом привязать фото к созданному `measurement.id`.

## Где хранятся фото

Файлы хранятся в Supabase Storage:

```text
bucket: measurement-photos
```

Записи о фото хранятся в таблице:

```text
measurement_photos
```

Обычно в записи фото есть:

- `measurement_id`;
- `photo_type`;
- `file_path`;
- `is_required`;
- `added_by`;
- дополнительные поля вроде имени файла и размера, если они есть в текущей схеме.

`file_path` связывает строку базы с объектом в Storage.

## `measurement_photos`

`measurement_photos` — это не сам файл, а запись-индекс для приложения.

Она нужна, чтобы:

- понять, к какому замеру относится фото;
- отфильтровать фото только текущего замера;
- показать тип фото;
- получить путь файла в Storage;
- удалить связь и файл при удалении фото/замера.

Если файл есть в Storage, но строка `measurement_photos` не создана, приложение считает загрузку ошибочной и пытается удалить загруженный файл.

## Signed URLs

Для просмотра фото приложение получает signed URL по `file_path`.

Это значит:

- Storage bucket может оставаться закрытым;
- в интерфейс попадает временная ссылка для просмотра;
- если signed URL истёк или файл удалён, превью может не открыться;
- в таком случае нужно обновить список фото или проверить наличие файла в Storage.

## Удаление случайно добавленного фото

В карточке замера у каждого фото есть кнопка удаления.

Для online-фото приложение:

1. Проверяет, что фото относится к текущему замеру.
2. Удаляет строку из `measurement_photos`.
3. Удаляет файл из Storage bucket `measurement-photos`.
4. Перечитывает фото текущего замера.

Если Supabase возвращает ошибку прав, нужно проверить RLS для `measurement_photos` и `storage.objects`.

## Offline-фото

Если открыт локальный TEMP-черновик, фото сохраняются не в Supabase, а в IndexedDB на телефоне.

Сохраняется:

- blob файла;
- локальный id;
- локальный id TEMP-черновика;
- тип фото;
- имя файла;
- MIME type;
- размер;
- статус синхронизации.

При синхронизации TEMP-черновика фото отправляются в Storage и получают server-поля.

## Если синхронизация offline-фото не прошла

Не удалять TEMP-черновик и локальные фото.

Что делать:

1. Проверить интернет.
2. Открыть TEMP-черновик.
3. Повторить синхронизацию.
4. Если сам замер уже создан, повторная синхронизация должна попытаться отправить оставшиеся фото.
5. Если ошибка повторяется, проверить:
   - права на Storage bucket `measurement-photos`;
   - RLS для `measurement_photos`;
   - размер фото;
   - наличие `server_id` у TEMP-черновика;
   - ошибку, сохранённую в локальном статусе фото.

Локальное фото можно удалить только если оно уже не нужно или точно отправлено в Supabase.
